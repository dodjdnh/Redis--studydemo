## 4. 三大缓存问题一句话区分

| 问题 | 一句话区分 | 本阶段解决方案 |
|------|------------|----------------|
| 缓存穿透 | 查一个根本不存在的数据，Redis 没有，数据库也没有 | 布隆过滤器拦截 + 空值缓存 |
| 缓存击穿 | 一个热点 key 刚好过期，大量请求同时打到数据库 | 互斥锁 + 双重检查 |
| 缓存雪崩 | 大量 key 同时过期，大量请求同时打到数据库 | TTL 随机抖动 + 分批预热 |

## 5. IM 场景对应关系

| IM 场景 | 可能的问题 | 推荐策略 |
|---------|------------|----------|
| 查询用户基础信息 | 恶意查不存在 userId，形成穿透 | 布隆过滤器判断 userId 是否可能存在；查不到写短 TTL 空值 |
| 查询群组基础信息 | 热门群组缓存过期，形成击穿 | 对 `group:info:{groupId}` 加重建锁和双重检查 |
| 查询会话成员关系 | 热门群聊成员缓存过期，ContactService 压力变大 | 对 `session:members:{sessionId}` 加互斥重建 |
| 服务启动预热用户/群组缓存 | 大量 key 同一时间过期，形成雪崩 | TTL = 基础时间 + 随机抖动 |
| 消息列表展示昵称头像 | 每条消息都查 Redis，网络开销高 | Caffeine 本地缓存 + Redis 远程缓存 |
| 权限类数据，例如是否在群、是否拉黑 | 本地缓存不一致可能造成越权 | 可以短 TTL 缓存，但必须谨慎，必要时删除通知或直接查权威服务 |

## 6. 与 RTC 项目现状的对比

| RTC 现状 | 风险 | 阶段三后的改进方向 |
|----------|------|--------------------|
| 发送消息前调用 `ContactSessionValidationClient` 远程校验 | 高并发下下游 ContactService 压力大 | 可以缓存会话成员或校验结果 |
| 当前没有用户信息/群组信息缓存 | 频繁展示昵称、头像、群名时可能重复查下游服务 | 用 Redis 缓存基础资料，热点数据可加 Caffeine |
| 当前没有穿透防护 | 非法 userId/sessionId 可能反复打到下游 | 加布隆过滤器或空值缓存 |
| 当前没有击穿防护 | 热门 session 缓存过期时可能大量请求重建 | 用 Redisson `RLock` 做缓存重建锁 |
| 当前没有雪崩防护 | 批量预热或统一 TTL 可能导致同时过期 | 给 TTL 增加随机抖动 |

## 7. 常见坑

### 7.1 空值缓存 TTL 不要太长

空值缓存用于防穿透，但如果 TTL 太长，会掩盖新创建的数据。

例如用户刚注册成功，但 Redis 里还有：

```text
user:info:10003 = __NULL__
```

如果这个空值缓存保留 1 天，系统可能 1 天内都认为用户不存在。

### 7.2 布隆过滤器说“可能存在”不等于真的存在

布隆过滤器的特点是：

```text
说不存在：一定不存在
说可能存在：可能存在，也可能误判
```

所以它不能替代数据库，只能作为第一道拦截。

### 7.3 抢到缓存重建锁后必须二次查 Redis

如果没有双重检查，排队等锁的请求会一个接一个查数据库。

正确流程是：

```text
抢到锁后，再查一次 Redis。
如果别人已经重建好缓存，直接返回。
```

### 7.4 `lock()` 不适合所有缓存重建场景

如果使用 `lock()`，抢不到锁会一直阻塞。

缓存重建更常用：

```java
tryLock(2, TimeUnit.SECONDS)
```

抢不到可以稍后重试或降级返回，避免请求线程堆积。

### 7.5 预热不能把所有 key 用同一个 TTL 写进去

批量预热时如果统一写：

```text
TTL = 30 分钟
```

30 分钟后可能集中失效。

应该使用：

```text
TTL = 30 分钟 + 随机 0~5 分钟
```

### 7.6 本地缓存会有多实例不一致

Caffeine 存在当前 JVM 内存里。

如果有多个 RTC 实例：

```text
RTC-1 本地缓存老昵称
RTC-2 本地缓存新昵称
```

短时间不一致是正常风险。对权限类数据要更谨慎。

### 7.7 VS Code 加依赖后要刷新 Maven 项目

本阶段加了 Caffeine 依赖。

如果启动时报：

```text
ClassNotFoundException: com.github.benmanes.caffeine.cache.Caffeine
```

说明运行 classpath 里没有 Caffeine jar，需要在 VS Code 里执行：

```text
Java: Clean Java Language Server Workspace
Maven: Reload All Maven Projects
```
